# 🚀 Game Engine Performance & Configuration Testing (Zuma)

## 📌 Project Overview
This repository demonstrates **Technical QA** skills through the manipulation of engine configuration files to perform stress testing and boundary analysis.

## 🛠️ Technical Implementation
I have refactored the `levels.xml` files for both Zuma Deluxe and Zuma's Revenge to validate engine stability under custom constraints.

### 🔍 Key Actions:
* **Global Parameter Override:** Systematically modified the `speed` attribute to a constant **0.50** across 130+ level instances to monitor engine behavior.
* **Stress Testing:** Pushed the game logic to 5x its default velocity to test collision detection and object synchronization.
* **Data Integrity:** Executed large-scale XML refactoring while maintaining schema validity and system functionality.

## 💼 QA Competencies
* **Back-end Validation:** Navigating and modifying complex data structures.
* **Boundary Value Analysis:** Testing system limits outside normal parameters.
* **Consistency:** Executing repetitive technical tasks with 100% accuracy.
